# 📧 Mail Organizer Skill

A modular system to automatically classify, archive, and route automated/corporate emails out of Thunderbird inboxes across multiple accounts (Gmail, Outlook, etc.) using the Thunderbird MCP bridge.

---

## 📋 Requirements

* **Operating System**: Linux (tested on Ubuntu/Debian sandboxed Flatpak and bare-metal environments).
* **Mail Client**: Mozilla Thunderbird (Version 115+ recommended) with the **Thunderbird MCP** extension installed.
* **Bridge Dependency**: Node.js (v18+) to run the transparent stdio-to-HTTP MCP bridge (`mcp-bridge.cjs`).
* **Python**: Python 3.8+ for executing the automation scripts.

---

## 🛠️ Walkthrough: Operating & Running the Skill

### 1. Identify Installation Context (Bare-Metal vs Flatpak)
Before running mail cleanups, identify if your Thunderbird is sandboxed:
* **Flatpak Detection**: Run `ps aux | grep -i thunderbird`. Look for Bubblewrap wrappers (`/usr/bin/bwrap`) and path locations inside `/app/lib/thunderbird` or `/run/user/1000/app/org.mozilla.thunderbird_esr`.
* **Bare-Metal Detection**: Will show standard local process binaries (e.g. `/usr/bin/thunderbird`) and write connection logs directly to `/tmp/thunderbird-mcp/`.

### 2. Verify if Thunderbird MCP is Running & Open
To ensure the script can talk to your mail client:
1. Ensure Thunderbird is open.
2. Check if the bridge process is listening: `ps aux | grep -i mcp-bridge` (should show `node ... mcp-bridge.cjs` running).
3. Find the autogenerated token file:
   * **Flatpak**: `/run/user/1000/app/org.mozilla.thunderbird_esr/thunderbird-mcp/connection.json`
   * **Bare-Metal**: `/tmp/thunderbird-mcp/connection.json`
4. Test bridge connectivity by checking if the port inside `connection.json` responds to a REST Ping:
   ```bash
   curl -X POST -H "Authorization: Bearer <YOUR_TOKEN>" http://127.0.0.1:<PORT>/
   ```

### 3. Run Senders Audit
Run the audit script to check what uncategorized corporate senders are cluttering the inbox:
```bash
python3 print_remaining.py
```

### 4. Adjust the Classification Rules
Add the identified domains/addresses to `move_inbox_emails.py` under the appropriate category inside `classify_message`:
```python
if any(domain in author for domain in ["mycompany.com", "service.io"]):
    return "Work"
```

### 5. Execute Inbox Cleanups
Run the main script to sweep, classify, and archive emails. The script will automatically discover all active IMAP/POP3 accounts and route emails:
```bash
python3 move_inbox_emails.py
```

## ⚙️ Generic Multi-Provider Architecture

The system avoids hardcoded folder paths so it can clean up Outlook, Gmail, and Yahoo accounts concurrently:
* **Dynamic Spam Resolution**: Discovers folder mappings by querying folder properties (e.g. `type == 'junk'` matches `[Gmail]/Spam` for Google and `Junk Email` for Outlook).
* **Dynamic Archive Fallback**: If a target category folder is not subscribed or refuses creation, the script queries folders with `type == 'archive'` to safely archive the message to `All Mail` (Gmail) or `Archive` (Outlook/other IMAP).
